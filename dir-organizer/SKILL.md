# dir-organizer — 安全なディレクトリ整理

散らかったディレクトリを、**中身を読んで意味的に分類**し、安全に整理するスキル。

## 大原則（必ず守る）

このスキルの価値は「壊さないこと」と「全部追えること」にある。ユーザーは過去に
「どこいったか分からなくなる」整理で困っている。だから:

1. **削除しない** — 不要ファイルはゴミ箱ではなく `_捨て/` ディレクトリへ移すだけ。
   最終的に消すかどうかはユーザーが自分で判断する。
2. **実行前に必ず確認** — `preview` で計画を提示し、日本語で要約してユーザーの承認を得てから
   `apply` する。勝手に動かさない。
3. **全移動を記録** — `apply` は manifest を残す。「あのファイルどこ行った?」に常に答えられ、
   `undo` で完全に元へ戻せる。
4. **取り残しを作らない** — 対象内の全ファイルが分類対象に入っているか確認する。

スクリプトは `scripts/organize.py`（Python 標準ライブラリのみ・依存ゼロ）。

## ふたつのモード

依頼内容に応じて使い分ける。どちらも scan → plan → preview → apply → undo の流れは同じ。

- **その場整理**: 1つの散らかったフォルダを、その中で分類する。plan の `from` は相対パス。
- **集約 + 重複排除**: 講義資料などが**複数の場所に散らばって重複している**のを、1か所に集めて
  同一内容の重複を1つに絞る。plan の `from` は絶対パス、`to` は宛先ルートからの相対パス。
  → 詳細は下の「集約モード」を参照。

## ワークフロー（その場整理）

### 1. 対象ディレクトリの確認
どのディレクトリを整理するか確定する。深い階層まで対象にするか（`--depth`）も確認するとよい。
危険な対象（`~` 直下、システムディレクトリ、Git 管理下の作業ツリー全体など）は範囲を狭めるか
ユーザーに確認する。

### 2. スキャン
```
python3 scripts/organize.py scan "<対象dir>" --out /tmp/scan.json
```
複数の場所をまとめて見る場合はディレクトリを並べる（集約モード）:
```
python3 scripts/organize.py scan ~/Downloads ~/Desktop "~/Documents/講義" --out /tmp/scan.json
```
`scan.json` には各ファイルの `abspath`・`path`（所属rootからの相対）・`root`・サイズ・更新日時・
拡張子・バイナリ判定・**先頭スニペット**・`sha1`・ゴミ候補フラグ・`course_hint`（講義資料らしさ）が入る。
`duplicates` には**場所をまたいで**同一内容（同 sha1）のファイル組が絶対パスで並ぶ。

### 3. 分類して plan.json を作る（ここが頭の使いどころ）
`scan.json` を読み、**スニペットと拡張子の両方**を見て各ファイルの行き先を決める。
分類カテゴリと「いる/いらない」の判定基準は `references/classification.md` を参照（必要時に読む）。

- **いるファイル** → 内容に合った意味的なフォルダへ。例: 仕様メモ→`ドキュメント/仕様/`、
  請求書PDF→`ドキュメント/請求書/`、スクショ→`画像/スクリーンショット/`、`.py`→`コード/`。
  拡張子が同じでも中身が違えば行き先を変える（スマート分類）。
- **いらないファイル** → `_捨て/` 配下へ。`junk` フラグが付いたもの（`.DS_Store`、`*.tmp`、
  0バイト等）に加え、内容を見て明らかに不要と判断したもの（古い一時出力、重複の片方など）。
  迷ったら「いる」に倒す（削除ではないが、隔離も慎重に）。

`plan.json` の形式:
```json
{
  "root": "<対象dirの絶対パス>",
  "trash_dir": "_捨て",
  "moves": [
    {"from": "古い仕様メモ.md", "to": "ドキュメント/仕様/古い仕様メモ.md", "reason": "仕様書の下書き"},
    {"from": "scan_tmp.log",    "to": "_捨て/scan_tmp.log",               "reason": "一時ログ・不要"}
  ]
}
```
`from`/`to` は対象dir からの相対パス。各 move に短い `reason`（日本語）を付けると、後で
ユーザーが判断を追える。**取り残しを無くしたい場合は全ファイルを moves に入れる**（残す
ファイルは現状維持なので move 不要だが、その場合 preview が「未分類」として警告する）。

### 4. プレビュー（dry-run）して確認を取る
```
python3 scripts/organize.py preview "<対象dir>" --in /tmp/plan.json
```
宛先別ツリー・件数・宛先衝突・**plan に含まれない取り残しファイル**を表示する（移動はしない）。
出力を読み、ユーザーに日本語で要約して提示する。例:

> 「`ドキュメント/` に 12 件、`画像/` に 8 件、`コード/` に 3 件を移動します。
>  `_捨て/` には `.DS_Store` など不要候補 5 件を隔離します（削除はしません）。
>  未分類で元の場所に残るファイルが 2 件あります。このまま進めてよいですか?」

承認が得られるまで `apply` しない。修正要望があれば plan.json を直して再 preview する。

### 5. 適用
```
python3 scripts/organize.py apply "<対象dir>" --in /tmp/plan.json --yes
```
移動を実行し、`<対象dir>/_整理ログ/manifest-<日時>.json` に全移動を記録する。
宛先が衝突した場合は上書きせず ` (1)` などを付けてリネームする。

### 6. 報告
- 何件をどこへ移したか、`_捨て/` に何件隔離したかを伝える。
- 「`_捨て/` の中身を確認して、不要なら自分で削除してください」と案内する。
- 「`undo` で全部元に戻せます」と必ず伝える:
  ```
  python3 scripts/organize.py undo "<対象dir>"
  ```

## 集約モード（複数の場所から集めて重複排除）

「講義資料があちこちにあって重複してる、まとめて整理したい」といった依頼向け。

1. **集める場所と宛先を決める**。どのフォルダを探すか（例: `~/Downloads`, `~/Desktop`,
   `~/Documents`）をユーザーに確認し、**集約先ルート**（例: `~/Documents/講義資料`）を決める。
   宛先ルートは新規でも既存でもよい。`_捨て/` と `_整理ログ/` はこのルート配下に作られる。
2. **複数ディレクトリを scan**（上記の通りディレクトリを並べる）。
3. **plan.json を作る**。`root` を宛先ルートにし、各 move の `from` は**絶対パス**（scan の
   `abspath`）、`to` は宛先ルートからの相対パスにする。
   - **重複排除**: `duplicates` の各組で、正本を1つ選んで講義フォルダへ。**残りは `_捨て/` へ**
     （削除ではなく隔離）。正本は「分かりやすい名前」「新しい更新日時」「元から正しい場所にある」
     ものを優先。
   - **集約先の構造**: 科目ごとに分けると探しやすい。例:
     `線形代数/第3回.pdf`、`ミクロ経済/講義05.pdf`。`course_hint` が True のものを中心に、
     スニペットや名前で科目を判断する。
   - 講義と無関係なファイルは **plan に入れない**（元の場所に残る）。集約モードでは宛先ルートの
     外にあるファイルは取り残し警告の対象外なので、無関係ファイルを誤って動かす心配はない。

   ```json
   {
     "root": "/Users/you/Documents/講義資料",
     "trash_dir": "_捨て",
     "moves": [
       {"from": "/Users/you/Downloads/第3回_線形代数.pdf", "to": "線形代数/第3回.pdf", "reason": "正本"},
       {"from": "/Users/you/Desktop/線形代数_第3回.pdf",   "to": "_捨て/線形代数_第3回.pdf", "reason": "上と同一内容の重複"}
     ]
   }
   ```
4. **preview → 確認 → apply → 報告 → undo** は その場整理と同じ。`apply`/`preview`/`undo` の
   位置引数には**宛先ルート**を渡す。undo は集めたファイルを元の散らばっていた場所へ戻す。

## undo（元に戻す）
最新 manifest を逆再生して、全ファイルを元の場所へ復元し、空になったカテゴリフォルダや
`_捨て/` を片付ける。集約モードでも、各ファイルは元々あった場所（絶対パス）へ戻る。
特定の manifest を指定するなら `--manifest <path>`。

## 注意
- スクリプトはどのカレントディレクトリからでも絶対/相対パスで呼べる。例では `scripts/organize.py`
  と書いているが、実際にはこの SKILL.md と同じ場所の `scripts/organize.py` を使うこと。
- 大量ファイルや巨大ファイルが多い場合、ハッシュ計算を省くなら scan に `--no-hash` を付ける。
- 物理削除は一切行わない。ユーザーが明示的に「`_捨て/` を消して」と言った場合のみ、内容を
  確認したうえで削除する。
