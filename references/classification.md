# 分類リファレンス

`scan.json` の各ファイルを `plan.json` の行き先に振り分けるための指針。
**拡張子だけで決めず、`snippet`（内容）も必ず見る**こと。同じ拡張子でも中身で行き先が変わる。

## カテゴリ体系（推奨）

トップレベルは日本語フォルダで分かりやすく。サブフォルダは内容に応じて作る。

| トップ | 入るもの | サブフォルダ例 |
|--------|----------|----------------|
| `ドキュメント/` | 文章・資料・PDF・Office | `仕様/` `請求書/` `契約/` `メモ/` `論文/` |
| `画像/` | png jpg gif svg webp heic | `スクリーンショット/` `写真/` `素材/` |
| `動画・音声/` | mp4 mov mp3 wav | `録画/` `音源/` |
| `コード/` | ソース・スクリプト・設定 | `python/` `js/` `shell/` `設定/` |
| `データ/` | csv json xml yaml sqlite | `csv/` `エクスポート/` |
| `アーカイブ/` | zip tar gz dmg pkg | — |
| `インストーラ/` | dmg pkg exe msi（一度使ったもの） | — |
| `_捨て/` | 不要ファイルの隔離（削除しない） | — |

カテゴリは対象ディレクトリの性質に合わせて調整してよい（例: 写真中心なら `画像/` を細かく、
開発フォルダなら `コード/` を言語別に）。過剰に細かくしない。

## 「いらない」と判断してよいもの（→ `_捨て/`）

scan が `junk: true` を付けるもの:
- システム生成: `.DS_Store`, `Thumbs.db`, `desktop.ini`, `.localized`
- 一時/バックアップ拡張子: `.tmp` `.temp` `.swp` `.bak` `.orig` `.crdownload` `.part`
- コンパイル中間物: `.pyc` `.pyo`
- Office ロック: `~$xxxx`
- 0バイトの空ファイル

内容を見て不要と判断してよいもの（慎重に）:
- 重複ファイルの片方（`duplicates` 組のうち、明らかに古い/コピー名のもの。`コピー`, `copy`,
  `(1)`, `最終_最終`, `final_v2` など）
- 一時出力・キャッシュらしきもの（`scan_tmp.log`, `output_old.txt` など、内容も使い捨て）

**迷ったら隔離しない**。`_捨て/` 行きも「移動」であり、ユーザーの確認対象。判断に迷うものは
通常カテゴリに入れて残す方が安全。

## 内容スニフの指針

- `binary: false` のファイルは `snippet` を読んで判断する。
  - Markdown/テキストの冒頭見出しや本文で用途を判定（仕様・メモ・README・ログ等）。
  - JSON/YAML/CSV はデータ → `データ/`。ただし設定ファイル（`package.json`,
    `tsconfig.json`, `.eslintrc` 等）は `コード/設定/`。
  - シェバン（`#!/usr/bin/env python` 等）やコード構文が見えたら `コード/`。
- `binary: true` は拡張子とファイル名で判断（画像/動画/アーカイブ/インストーラ）。
- 拡張子が無い・不明なファイルはスニペットで判定。判定不能なら残す（move しない）。

## 重複の扱い（集約モード）

`scan.json` の `duplicates` は**同一内容（同 sha1）**のファイル組。サイズやハッシュが同じなら
中身は同一なので、安全に1つへ絞れる。

- 各組から**正本を1つ**選び、適切なフォルダへ移す。残りは `_捨て/` へ隔離（削除しない）。
- 正本の選び方（優先順）:
  1. すでに正しい/恒久的な場所にあるもの（`~/Documents/講義/...` など）
  2. 名前が分かりやすいもの（`線形代数_第3回.pdf` > `スキャン_001.pdf`、`(1)` や `copy` が付かない）
  3. 更新日時が新しいもの（`mtime`）
- **名前だけ違って中身が同じ**ものが重複。中身が少しでも違えば sha1 が変わり別ファイル扱いに
  なるので、`duplicates` に出ていないものは内容を見て個別に判断する（versionの違い等）。

## 講義資料の集約

`course_hint: true` は講義・授業資料らしいファイルの目印（パスやファイル名のキーワード判定）。
最終判断はスニペットと名前で行う。集約先は**科目ごと**に分けると後で探しやすい。

- 科目フォルダ例: `線形代数/`, `ミクロ経済/`, `データ構造/`。
- 回ごとの資料は `第3回.pdf` のように回数で揃えると並びが見やすい。
- 講義資料と一緒にあるが無関係なもの（個人メモ、ダウンロードしたインストーラ等）は plan に
  入れず元の場所に残す。集約は「関係するものだけ集める」操作。

## 名前付けのルール

- `to` のファイル名は原則そのまま維持する（リネームは混乱の元）。
- 宛先衝突はスクリプトが自動で ` (1)` を付けて回避するので、plan 側で無理に変えない。
- 日本語フォルダ名で問題ない（macOS / GitHub どちらも可）。
